## Problem Statement

Cloud-dependent multiplayer platforms such as Kahoot, Mentimeter, and Jackbox are built on the assumption of a stable, high-throughput internet connection. In practice, this assumption collapses in high-density environments — college auditoriums, rural classrooms, underground hackathon venues — where a single consumer-grade Wi-Fi router cannot handle the simultaneous HTTP polling and WebSocket traffic generated by 50 or more connected devices. The result is a cascade of timeouts, disconnections, and a ruined event.

The root cause is architectural: these platforms route every packet through a remote cloud server, introducing unnecessary round-trip latency and making the local network a single point of failure.

##  The Solution: An Infrastructure-Agnostic Engine

LocalFlux is not just a game; it is an open-source, self-hosted multiplayer framework. It decouples gameplay from ISP stability by treating the **Local Area Network (LAN)** as a first-class citizen, delivering 0ms latency gameplay entirely offline, while utilizing smart tunneling to prevent router hardware crashes.

## The Solution

**LocalFlux** is an open-source, self-hosted, offline-first multiplayer quiz engine. It eliminates the dependency on remote cloud infrastructure by treating the **Local Area Network (LAN)** as the primary transport layer. The host machine becomes the server. Players connect directly to it via WebSocket over the local network, achieving near-zero latency gameplay that is completely independent of ISP connectivity or router bandwidth to the internet.

LocalFlux is not simply a game — it is a portable, content-agnostic event engine that any organizer, teacher, or developer can deploy from a single machine with no cloud accounts, no subscriptions, and no internet connection required.

No external database is used at any point. All game state, session data, and match telemetry live entirely in memory and on the local file system.

## Architecture

### 1. LAN Binding and Local Routing

On startup, the Node.js server programmatically extracts the host machine's physical IPv4 address (e.g., `192.168.x.x`) using the `os` module and binds the Express and Socket.io server to that interface. Every player on the same Wi-Fi network connects directly to this address. No traffic leaves the local network.

### 2. In-Memory Game State

There is no database write on the hot path. All active game sessions, room state, player scores, and round timers are managed entirely in RAM using JavaScript `Map()` objects. This eliminates I/O latency and makes the engine fast enough to handle simultaneous buzzes from 50+ players with deterministic ordering.

### 3. Base64 Asset Delivery over WebSocket

To prevent players from pre-fetching question images via HTTP directory scraping, **LocalFlux does not serve static image URLs**. Instead, at the precise moment a round begins, the Node.js server reads the corresponding `.webp` file from the protected `data/vault/` directory using the `fs` module, encodes it as a Base64 string, and emits it directly to all connected clients through the Socket.io event stream. Players receive the image simultaneously with the question. There is no public asset endpoint to enumerate.

### 4. QR Gateway for Instant Join

The host screen displays a QR code that encodes the server's local IP address and port. Players scan it with any smartphone camera and are taken directly to the React client — no app download, no URL typing, no account creation required.

### 5. Deck System (Content-Agnostic Engine)

LocalFlux separates the engine from its content. A "deck" is a JSON file in `data/decks/` paired with a folder of optimized WebP images in `data/vault/`. Organizers and developers can create entirely new trivia categories — films, science, history, geography — by dropping a new deck file into the directory. No code changes are required. The engine parses and loads the deck at runtime.

### 6. Deck Studio (In-Browser CSV Parser)

A built-in React UI allows organizers to upload a `.csv` file containing questions and answers. The file is parsed entirely in the browser using native JavaScript — no server upload, no third-party library. The output is a valid deck JSON file ready to be placed in `data/decks/`.

### 7. Black Box Telemetry

At the conclusion of every match, the Node.js server writes a structured JSON log to the `logs/` directory containing the final leaderboard, per-question accuracy statistics, and session metadata. This gives event organizers a persistent record without requiring any database infrastructure.

## Getting Started

### Prerequisites

- Node.js v18 or higher
- npm v9 or higher
- Python 3.9+ (only required for running the data pipeline scripts)

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/Unknownbeliek/foss-hack-quiz-engine.git
cd foss-hack-quiz-engine
```

**2. Install server dependencies**

```bash
cd server
npm install
```

**3. Install client dependencies**

```bash
cd ../client
npm install
```

### Running in Development

**Start the backend server** (from the `server/` directory):

```bash
npm run start
```

The server will detect your local IP, bind to it, and print the LAN address and QR code URL to the terminal.

**Start the frontend development server** (from the `client/` directory):

```bash
npm run dev
```

Open the printed LAN address on any device connected to the same Wi-Fi network to join as a player, or open `localhost` to access the host dashboard.

### Production Build

```bash
# Build the React client
cd client
npm run build

# Start the server (serves the built client statically)
cd ../server
npm run start
```

## Significance and Impact

LocalFlux addresses a real and recurring failure mode at events that take place in infrastructure-constrained environments. By removing the cloud from the critical path entirely, it makes interactive multiplayer experiences accessible in settings where they were previously unreliable or impossible.

Because it is fully open source and content-agnostic, LocalFlux is equally useful to a hackathon organizer running a film trivia night, a schoolteacher conducting a classroom quiz with no internet, or a developer who wants a foundation for building their own custom multiplayer game. The deck system means anyone can author and share new content without writing code.

The absence of any external dependency — no database, no cloud account, no paid service — means the total cost of running LocalFlux at any scale is zero.
